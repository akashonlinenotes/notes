OOP i.e Object Oriented Programming is a programming methodology that helps organize complex programs through the use of inheritance, encapsulation, and polymorphism.

Before Object Oriented Programming, Structured Programming was prevelant (languages like C were based on structured programming). While structured programming enabled programmers to write moderately complex programs very easily, but when the size of the program reaches a certain size, its complexity exceeded what a programmer can manage. Therefore a need for a way of programming that can help managing large volumes of code effeciently was felt. The answer to this problem was Object Oriented Programming.

C was based on structured programming and the code was difficult to manage as the size of the program grew.
To solve this problem, C++ was invented which allowed organizing big peices of code through the use of classes. (C++ was initially named C with Classes).
The problem with languages like C and C++ was, they were not platform independent or architecture neutral which meant that the code written using C/C++ could be compliled using only specific compilers. This meant that, the compiler required within a microwave oven and remote control should be the same.A language was needed which could be compiled by any type of complier. Java was the solution. Code written in java can be plugged into remote ccontrol and microwave owen without worrying about the compilers used internally.

WWW (World Wide Web). Success of WWW needed portable programs (programs that can be executed on any ennvironment) and Java was the only solution that time.

An applet is a special kind of Java program that is designed to be transmitted over the internet (from server to client) and gets automatically executed inside java enabled web browser on the client side.

1. Java gave applets for use over the internet:- Java applets are java programs that were initiated from the server side and were executed on the client side. They were mostly used to display the information retreived from the server to the client.

2. Java programs promised security to client machine when downloaded from internet:- The program/application which the user downloads on his machine can get access to the system resrouces and send back confidential info over the network to designated servers. To prevent this security breach, java allowed java applications/programs downloaded over the internet on client machine to be confined only to the Java execution environment and prevented the java programs from accessing other parts of the computer.

3. Java programs guaranteed executiion on any environment (comnination of OS, CPU's architecture etc):- The program transmitted from server over the internet to the client computer must get executed on it regardless of what environment it has i.e the same application code should work on all computers connected to the internet. The programs written in java can be executed on any environment and therefore java became the most popular programming language of the WWW.

Byte code is highly optimized set of instructions which is executed by JVM (which is part of JRE)

Byte code

In most programming languages, the source code is directly converted into executable code by the compiler. However, in case of java, on compiilation the java source code is first converted to an intermediate form called the byte code which is later converted into the executable code.

JVM, JRE

Then different versions of the same java program need to be created for executing it on each different type of CPU connected to the internet.

1. The presence of byte code makes java programs portable:- Once the source code is converted to byte code, this byte code can be executed on any environment that has JVM/JRE installed. If byte code was not there, then different versions of the same program need to be created to run them on differing environment configurations.

2. The presence of byte code makes java programs secure to execute:- Byte code is executed by JVM. Before JVM executes the byte code, it creates a restricted execution environment (called sandbox) which does not allow the java program (byte code) to gain access to other system resources there by ensuring security of the system on which the java programs are executed.

The performace of Java is slightly compromised. The time taken by java programs for converting the java source code into executable code is slightly higher than those programming languages in which the source code gets directly compiled to executable code.

No, after the java source code is converted to byte code by java compiler, the conversion from byte code to executable code happens much faster than normal. This happens due to the presence of JIT (Just-In-Time compiler) in JVM (which is part of JRE) which does not convert all the byte code into executable code at once, instead convertion of byte code to executable code happens piecce by piece as needed during execution. JIT provides significant performance boost to Java

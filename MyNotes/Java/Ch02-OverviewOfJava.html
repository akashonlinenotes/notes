main() method

The main() method is always public. It is because main() method is called by JVM when the Java Application Launcher is invoked and public keyword grants access to any code outside the class in which main() method is present to access/call the main() method. Private or Protected cannot be used with main() method.

The main() method is called by JVM when the Java Application Launcher is invoked. JVM does not instantiate the class in which the main() method is defined to access the main() method. The main() method is simply accessed by JVM using the class name on which the Java Application Launcher is invoked like Classname.main(). It is therefore mandatory for main() method to be static.

The caller of the main() method is JVM. There is no point of returning any value to the JVM since the user does not have access to the JVM code. Therefore the return type of main() method is void.

1. On compilation, no error is reported and the file gets compiled successfully.

2. On execution, error: "Main method must return a value of type void" gets displayed.

Comments are the statements which the programmer adds to the code to explain what the code is doing.

The characterisitic property of comments is that, they are simply ignored by the compiler.

In java, there are 3 types of comments:-

1. Single line comments:- They begin with two forward slashes and end with the line. They are used to comment only one line.

2. Multi line comments:- They begin with a forward slash and a asterisk, extend over multiple lines and end with a asterisk and a forward slash. They are used to comment multiple lines together. Note:- Multi line comments can also be used in a single line but using like that does not make sense when single line comments are already there for the purpose.

3. Documentation comments:- They begin with a forward slash and two asterisk symbols, stretch over multiple lines and end with an asterisk symbol and a forward slash. They are used to generate method documentation.
